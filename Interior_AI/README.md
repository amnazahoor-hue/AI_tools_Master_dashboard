# Interior-AI

Professional virtual staging web app built with Flask + React.

Interior-AI lets you upload an empty room photo, apply a design theme, and generate an AI-staged result with a before/after comparison slider and downloadable output.

## Tech Stack

- Backend: Flask, Pillow, Requests
- Frontend: React (Vite), Tailwind CSS, Framer Motion
- AI Provider: Pollinations.ai (`flux` model)

## Features

- Theme-based virtual staging (Minimalist, Industrial, Bohemian, Modern Office)
- Drag-and-drop + click upload
- Client-side image compression before upload
- Polished 3-phase generation stepper
- Before/After interactive transformation slider
- Download staged image
- UUID-based storage for original/staged assets

## Prerequisites

- Python 3.10+
- Node.js 18+ and npm
- Internet access (Pollinations image generation is remote)

## Environment Setup

### Backend `.env`

Create `backend/.env`:

```env
POLLINATIONS_API_KEY=your_pollinations_api_key_here
PORT=5000
```

> Note: `HF_API_TOKEN` is no longer used.

## Installation

### 1) Backend

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

### 2) Frontend

```powershell
cd ../frontend
npm install
```

Optional: create `frontend/.env`:

```env
VITE_API_BASE_URL=http://localhost:5000
```

Open 2 terminals.

## API

### `POST /api/stage-room`

Multipart form fields:

- `image` (required): room photo
- `theme` (optional): one of supported themes
- `prompt_strength` (optional): retained for compatibility

Response:

```json
{
  "id": "uuid",
  "theme": "Minimalist",
  "prompt_strength": 0.55,
  "original_image_url": "/api/uploads/<uuid>_original.jpg",
  "staged_image_url": "/api/uploads/<uuid>_staged.png",
  "download_url": "/api/download/<uuid>_staged.png"
}
```

## How AI Generation Works

1. Backend preprocesses uploaded image with Pillow (orientation fix, RGB normalize, resize).
2. Backend builds a theme prompt (architectural, photorealistic style).
3. Backend calls Pollinations endpoint:
   - `https://image.pollinations.ai/prompt/{prompt}?width=1024&height=1024&nologo=true&seed=<random>&model=flux`
4. Generated bytes are saved to `backend/uploads/` with UUID names.

## Troubleshooting

- `Failed to fetch`:
  - Confirm backend is running on `:5000`.
  - Confirm `VITE_API_BASE_URL` points to backend URL.
- 400 error from `/api/stage-room`:
  - Check `backend/.env` has valid `POLLINATIONS_API_KEY`.
  - Check internet connectivity and provider availability.
- CORS issues:
  - Backend already enables CORS on `/api/*`.
